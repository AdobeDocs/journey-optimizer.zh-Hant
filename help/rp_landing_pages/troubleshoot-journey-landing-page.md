# 監視與疑難排解{#section-overview}

若要有效管理您的客戶歷程，則需要主動監視功能、深入的報告功能，以及有效的疑難排解功能。Adobe Journey Optimizer 提供全面的工具，用於追蹤歷程績效、識別潛在問題，並在問題發生時快速解決問題。

使用[系統警示](../using/reports/alerts.md)接收有關嚴重事件的通知，利用[歷程報告](../using/reports/journey-global-report-cja.md)分析績效度量，並檢閱[歷程步驟事件](../using/reports/journey-step-events-overview.md)以了解詳細的執行流程。發生問題時，無論您是遇到歷程失敗、執行不一致還是動作層級的問題，請查閱[錯誤碼](../using/building-journeys/error-codes-reference.md)以取得快速解決方案，並遵循特定問題區域的目標疑難排解指南。

此頁面彙集了基本資源，可幫助您監視歷程健康情況、存取關鍵報告深入分析，以及有效解決客戶歷程所有階段的問題。

## 報告與監視

存取關鍵報告以監視歷程績效、分析量度並識別問題：

* [歷程全域報告](../using/reports/journey-global-report-cja.md) - 檢視跨所有管道的全面歷程績效量度和分析。

* [歷程即時報告](../using/reports/journey-live-report.md) - 監視即時歷程執行資料並追蹤進行中的客戶互動。

* [歷程步驟事件](../using/reports/journey-step-events-overview.md) - 分析詳細的逐步歷程執行事件，以進行疑難排解和最佳化。

* [報告快速入門](../using/reports/report-gs-cja.md) - 了解報告功能、量度，以及如何存取歷程深入分析。

* [傳遞能力監視](../using/reports/deliverability.md) - 追蹤訊息傳遞率，並識別跨管道的潛在傳遞問題。
