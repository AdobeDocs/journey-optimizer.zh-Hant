# 角色與職責

Adobe Journey Optimizer 可讓品牌在整個客戶歷程中提供連線、內容相關且個人化的體驗。Journey Optimizer 以端到端的規模、速度和靈活性為中心打造，將三個主要的價值驅動因素結合在統一的應用程式中：

* **即時客戶洞察和參與**，由 Adobe 的即時客戶輪廓提供支援
* 透過統一畫布 (針對即時歷程和批次行銷活動) 以及現代訊息設計工具進行&#x200B;**現代全管道協調**
* 透過決策管理和 AI/ML 功能進行&#x200B;**智慧決策與個人化**

Journey Optimizer 提供兩種協調方法，以符合不同的行銷需求：

* **歷程**：最適合即時、一對一的互動，每個客戶以自己的步調前進，由行為或事件觸發
* **協調的行銷活動**：最適合批次、一對多的行銷活動，透過排程的多步驟工作流程讓多個客群保持同一進度，適用於季節性促銷活動、產品推出和帳戶型通訊

此統一體驗可讓您在一個地方實作整個使用案例，從定義客群和設計歷程到建立個人化內容和分析結果。本文件說明有效使用 Journey Optimizer 的主要角色、其職責以及如何開始使用。

**重要注意事項：** Adobe Journey Optimizer 定義具有特定職責的不同角色。視您的組織結構而定，個人可以執行多個角色或所有角色。

## 角色型快速入門手冊

為簡化實作，Adobe Journey Optimizer 會根據專業知識將工作整理成特定角色。每個角色都專注於提供順暢客戶體驗所需的基本工作。

| 角色 | 主要職責 | 關鍵技能 | 典型工作 |
|-------------------|----------------------------------|--------------------------------|-----------------------------------------------|
| **管理員** | 環境設定和存取權管理 | 系統設定、使用者管理、安全性 | 設定沙箱、管理使用者權限、設定管道和訊息預設集 |
| **資料工程師** | 客戶輪廓資料與資料來源 | 資料模式、XDM 結構描述、來源連接器 | 將輪廓和業務資料模型化為結構描述、設定來源連接器、監視資料擷取 |
| **開發人員** | 技術實作與整合 | Mobile/Web SDK、API、事件導向架構 | 整合 SDK、實作事件、建立自訂動作端點 |
| **行銷人員** | 歷程設計和個人化體驗 | 歷程協調、內容建立、客群目標定位 | 設計客戶歷程、建立及個人化訊息、管理產品建議和決策元件、定義客群 |

每個角色都會處理 Adobe Journey Optimizer 實施的特定階段，並確保結構化且有效的部署流程。

## 實作順序和角色相依性

成功的 Journey Optimizer 實施通常會依照此順序進行，這會反映角色之間的相依性：

1. **管理員**：設定環境\
   管理員可以設定沙箱、設定存取控制及準備管道設定，進而奠定基礎。必須先執行此動作，才能讓其他團隊開始工作。
   * 設定開發、中繼和生產沙箱
   * 設定角色、權限和物件層級存取控制 (OLAC)
   * 設定管道設定 (電子郵件、簡訊、推播、應用程式內、網頁、內容卡)
   * 委派子網域並設定 IP 集區
   * 設定禁止名單和同意原則

2. **資料工程師**：建立資料基礎\
   資料工程師會建立支援個人化的資料基礎結構，定義客戶資料如何傳入及流經系統。
   * 建立用於客戶識別的身分命名空間
   * 設計 XDM 結構描述 (輪廓、體驗事件、關聯式)
   * 設定資料集並為即時客戶輪廓啟用資料集
   * 設定資料擷取 (批次和串流)
   * 建立複雜計算的計算屬性
   * 設定歷程的事件和資料來源

3. **開發人員**：實作技術整合\
   開發人員透過整合 SDK、傳送事件及建立 API 端點，將應用程式連線至 Journey Optimizer。這些實作可讓歷程觸發和執行。
   * 整合 Mobile SDK (iOS/Android) 與推播通知設定
   * 為網頁體驗實作 Web SDK
   * 從應用程式傳送事件以觸發歷程
   * 建立外部系統整合的自訂動作端點
   * 使用 Adobe Experience Platform Assurance 測試實作

4. **行銷人員**：設計和執行客戶體驗\
   行銷人員運用所有基礎工作來建立歷程、建立內容，並跨所有管道最佳化客戶體驗。
   * 使用細分、CSV 上傳或客群構成建立客群
   * 使用 AI 助理和範本設計個人化內容
   * 使用事件和客群觸發程序建立多管道歷程
   * 啟動前使用核准工作流程進行測試
   * 根據報告深入分析監視績效並最佳化

**注意：**&#x200B;雖然此順序是典型的，但某些活動可以並行發生。例如，開發人員可在資料工程師設定結構時進行應用程式整合。

## 角色快速入門

每個角色都從根據其重點量身打造的特定任務開始。完成這些初始步驟可使引導更順利完成，確保與整體實施流程保持一致：

### 針對行銷人員 {#for-marketers}

身為行銷人員或業務人員，您需要設計客戶歷程，以跨所有接觸點提供個人化的內容體驗。您將在統一的介面中工作，以從頭到尾實施整個使用案例。

**您將使用的關鍵功能：**

* **歷程協調**：建立即時、一對一的客戶參與，每個人以自己的步調前進，由行為或跨管道事件觸發
* **行銷活動協調**：使用視覺畫布設計和自動化大規模的複雜多步驟批次行銷活動。非常適合品牌啟動的行銷活動，例如季節性促銷活動、產品推出和帳戶型通訊。運用多實體細分，將客戶資料與相關實體 (帳戶、購買、預訂) 連線，以建立精確的客群
* **現代訊息設計工具**：使用拖放介面設計並個人化電子郵件和行動訊息。編輯現成的範本，以縮短上市時間
* **決策管理**：在可嵌入電子郵件和客戶接觸點的集中式資料庫中，建立和管理產品建議、適用性規則和其他元件。
* **資產管理**：存取完整嵌入 Journey Optimizer 中的 Adobe Experience Manager Assets Essentials，以簡化資產存取和傳送
* **客群定義**：使用關聯式查詢建立隨選客群並即時調整，在傳送前了解準確的客群計數
* **AI/ML 服務**：運用傳送時間最佳化和預測性參與分數，鎖定高價值客戶並將流失風險降至最低

**入門：**&#x200B;使用案例範本和精靈，以輕鬆建立及部署新的客戶歷程。

[行銷人員快速入門 →](path/marketer.md)

### 針對資料工程師 {#for-data-engineers}

身為資料架構師或工程師，您需要設定和維護客戶輪廓資料，以及其他支援 Journey Optimizer 所協調體驗的資料來源。

**主要職責：**

* **客戶輪廓資料**：將客戶輪廓資料和業務資料模型化為結構描述，以建立客戶的統一 360 度檢視
* **關聯式資料模式**：針對協調的行銷活動，設計關聯式結構描述以啟用多實體細分，將客戶資料與相關實體 (如帳戶、購買、訂閱和預訂) 連線，以建立彈性客群
* **來源連接器**：設定來源連接器，將資料從網頁、CRM、離線資料和其他來源擷取至 Adobe Experience Platform
* **身分解析**：設定身分命名空間以持續更新輪廓，並即時將客戶移入和移出區段和歷程
* **資料來源**：設定資料來源，以便在客戶歷程中即時監聽外部訊號
* **輪廓管理**：啟用即時客戶輪廓的資料集，以支援個人化體驗
* **資料品質**：監視資料擷取，以確保所有資料順利流入 Journey Optimizer

**入門：**&#x200B;模型化您的第一個客戶輪廓結構描述，並設定來源連接器以開始擷取資料。

[資料工程師快速入門 →](path/data-engineer.md)

### 針對管理員 {#for-administrators}

身為管理員，您需要設定 Journey Optimizer 環境，讓您的團隊有效率且安全地工作。

**主要職責：**

* **沙箱**：建立和管理沙箱，以分割不同使用者群組 (開發、測試、生產) 的資料和歷程
* **使用者管理**：設定使用者群組和權限，以控制不同功能的存取
* **管道設定**：設定傳遞管道和訊息預設集，以確保透過 Journey Optimizer 傳遞的訊息和資產之間保持一致的品牌形象
* **安全性與治理**：套用物件層級存取控制 (OLAC)、設定同意原則，以及實作資料治理原則
* **傳遞能力**：委派子網域、建立 IP 集區，以及管理禁止名單和允許清單
* **歷程設定**：為您的團隊設定歷程元素和設定

**入門：**&#x200B;設定沙箱和使用者權限，然後設定您的第一個管道設定和訊息預設集。

[管理員快速入門 →](path/administrator.md)

### 針對開發人員 {#for-developers}

實作技術整合，將 Journey Optimizer 連線至您的應用程式。

**主要職責：**

* 整合 Adobe Experience Platform Mobile SDK (iOS/Android)
* 實施適用於網頁體驗和網頁推播通知的 Web SDK
* 設定推播通知認證和憑證
* 從應用程式傳送事件以觸發歷程
* 建立 Journey Optimizer 透過自訂動作呼叫的 API 端點
* 針對網頁、行動裝置和其他表面實作程式碼型體驗
* 使用 Adobe Experience Platform Assurance 測試實作並偵錯
* 使用 Journey Optimizer API 以進行程式化存取

**入門：**&#x200B;整合 Mobile 或 Web SDK，然後實作您的第一個事件以觸發歷程。

[開發人員快速入門 →](path/developer.md)

## 跨角色共同作業

成功的 Journey Optimizer 實作需要跨所有角色共同作業。每個角色都與其他角色合作，提供順暢的客戶體驗：

>[!BEGINTABS]

>[!TAB 管理員]

**管理員**&#x200B;藉由管理存取權和設定來啟用所有團隊。他們會與下列人員合作：

* **資料工程師**：授予資料管理的權限、核准沙箱存取權、協調治理原則
* **開發人員**：提供 API 認證、設定測試環境、核准管道設定
* **行銷人員**：指派歷程/行銷活動的權限、設定管道、支援測試環境

>[!TAB 資料工程師]

**資料工程師**&#x200B;為所有人提供資料基礎。他們會與下列人員合作：

* **管理員**：請求資料管理的權限、協調治理和保留原則
* **開發人員**：提供 XDM 結構描述和事件結構、定義事件承載格式、測試資料擷取
* **行銷人員**：建立個人化的計算屬性、建立客群、設定關聯式結構描述

>[!TAB 開發人員]

**開發人員**&#x200B;實作技術整合以支援歷程。他們會與下列人員合作：

* **資料工程師**：取得 XDM 結構描述和事件結構，在資料收集規定，測試事件傳遞各方面達成共識
* **管理員**：提供 API 規格、請求權限和認證、協調測試策略
* **行銷人員**：瞭解事件觸發程序、實施追蹤、支援歷程測試、疑難排解問題

>[!TAB 行銷人員]

**行銷人員**&#x200B;設計客戶體驗，同時提供回饋意見。 他們會與下列人員合作：

* **資料工程師**：要求計算屬性、協調客群需求，再針對資料品質提供回饋意見
* **開發人員**：在事件觸發程式、測試實作、驗證追蹤等方面達成共識
* **管理員**：請求頻道設定、確認功能存取、協調賦能

>[!ENDTABS]

**最佳做法：**&#x200B;定期舉行跨職能會議，以便調整優先順序、共用進度，並處理跨團隊的封鎖程式。
