# Adobe Journey Optimizer套件和功能 {#ajo-packages}

>[!BEGINSHADEBOX]

**在此頁面上：**&#x200B;逐步進行模組化Adobe Journey Optimizer封裝、選擇基本選件、新增管道和新增進階功能，讓您可以依照與客戶互動的方式搭配正確的設定。

>[!ENDSHADEBOX]

[!DNL Adobe Journey Optimizer]使用模組化封裝模型。 從符合主要協調使用案例的基本優惠方案開始，然後新增您需要的頻道和進階功能。

>[!NOTE]
>
>可用功能可能會因您的授權合約、選取的附加元件、地區可用性、通道設定及實作先決條件而有所不同。 請聯絡您的Adobe代表，瞭解貴組織的特定詳細資訊。

## 步驟1 — 選擇基本選件 {#base-offers}

提供三個基本選件。 選擇最符合您與客戶互動方式的。

| 基本優惠方案 | 最適合 | 包含的內容 |
|-----------|---------|-----------------|
| **Journey Optimizer — 行銷活動** | 批次、行銷人員規劃的外聯活動 | 以對象為基礎、排程的行銷活動策劃，用於批次參與和行銷人員規劃的外聯活動。 |
| **Journey Optimizer — 歷程** | 即時客戶參與 | 事件導向，1:1歷程協調，用於即時客戶參與。 |
| **Journey Optimizer — 行銷活動和歷程** | 同時需要兩者的客戶 | 在一個選件中包含以對象為基礎的行銷活動協調和即時歷程協調。 |

### 行銷活動與歷程 — 有何差異？ {#campaigns-vs-journeys}

| 功能 | Journey Optimizer — 行銷活動 | Journey Optimizer — 歷程 | Journey Optimizer — 行銷活動和歷程 |
|--|:-----------------------------:|:----------------------------:|:----------------------------------------:|
| 受眾型批次協調 | ✓ | — | ✓ |
| 即時事件導向式協調 | — | ✓ | ✓ |
| 可用的管道附加元件 | ✓ | ✓ | ✓ |
| 可用的決策附加元件 | ✓ | ✓ | ✓ |

### 何時應選擇「行銷活動」？ {#when-to-choose-campaigns}

如果您的主要使用案例為：，請選擇&#x200B;**Journey Optimizer — 促銷活動**

- 排程的行銷活動
- 對象型外聯
- 批次行銷工作流程
- 行銷人員計畫的參與計畫

### 何時應選擇歷程？ {#when-to-choose-journeys}

如果您的主要使用案例為：，請選擇&#x200B;**Journey Optimizer — 歷程**

- 事件觸發式通訊
- 後續事件追蹤流程
- 入門和生命週期歷程
- 即時1:1參與

### 何時應選擇行銷活動和歷程？ {#when-to-choose-both}

選擇&#x200B;**Journey Optimizer — 行銷活動和歷程** （如果您同時需要）：

- 批次受眾型協調，以及
- 即時事件導向式協調

## 常見問題 {#faq}

+++**每個基本優惠方案是否包含每個頻道？**

沒有。 基本選件可定義您的協調功能。 管道存取權取決於貴組織的附加元件和授權設定。

+++

+++**行銷活動與歷程之間有何差異？**

**行銷活動**&#x200B;是以對象為基礎，且是行銷人員規劃的。 它們最適合用於排程的外聯活動、促銷方案和批次行銷活動工作流程。

**歷程**&#x200B;為即時和事件導向。 它們最適合1:1參與，當客戶行為發生時反作用。

**行銷活動和歷程**&#x200B;包含兩種協調模式。

+++

+++**協調的行銷活動支援哪些管道？**

協調的行銷活動支援行銷活動協調工作流程所使用的頻道子集。 管道可用性取決於您的授權設定。 [進一步瞭解協調的行銷活動](../orchestrated/gs-orchestrated-campaigns.md)

+++

+++**每個設定中是否包含決策？**

沒有。 Decisioning是一個獨立的進階功能附加元件。 [瞭解如何使用決策](../experience-decisioning/gs-experience-decisioning.md)

+++

+++**我聽說過Select、Prime或Ultimate。 這些仍然是目前的封裝模式嗎？**

[!DNL Adobe Journey Optimizer]現在透過以基本選件和附加元件為基礎的模組化封裝模型提供。 如果您是使用舊版封裝術語的現有客戶，請聯絡您的Adobe代表以確認您目前的權益。

+++
