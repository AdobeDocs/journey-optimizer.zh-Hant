# Adobe Journey Optimizer套件和功能 {#ajo-packages}

>[!BEGINSHADEBOX]

**在此頁面上：**&#x200B;瞭解模組化Adobe Journey Optimizer封裝如何跨基本優惠方案、通路附加元件和決定附加元件運作，以便您選擇符合參與使用案例和預算的組合。

>[!ENDSHADEBOX]

[!DNL Adobe Journey Optimizer]使用模組化封裝模型。 從符合您主要使用案例的基本優惠方案開始，然後新增您需要的頻道和進階功能。

>[!TIP]
>
>**不確定您使用哪個模型？** 如果您以模組化封裝模式購買[!DNL Adobe Journey Optimizer]，則本頁說明的基本選件和附加元件適用於您。 如果您看到合約中參考的&#x200B;**Select**、**Prime**&#x200B;或&#x200B;**Ultimate**，表示您使用舊版封裝模式 — 請連絡您的Adobe代表以確認您目前的權益。

>[!NOTE]
>
>套件可用性和包含的功能可能會因您的合約、選取的附加元件和地區可用性而有所不同。 請聯絡您的Adobe代表，瞭解貴組織的特定詳細資訊。

## 快速解答 {#quick-answers}

在進一步閱讀之前，大多數人都會遇到兩個問題。 [完整常見問題集](#faq)位於頁面底部。

+++**每個基本優惠方案是否包含每個頻道？**

沒有。 [!DNL Adobe Journey Optimizer]使用模組化模型：基本優惠方案會決定您的協調功能（行銷活動、歷程或兩者），而頻道附加元件會決定您可以參與的傳訊介面。 您可以選擇適合使用案例和預算的組合。

+++

+++**行銷活動與歷程之間有何差異？**

**行銷活動**&#x200B;是以對象為基礎，由行銷人員規劃 — 您可以定義對象、建立訊息，並以批次傳送方式排程或觸發。 它們最適合用於促銷外聯、電子報和多步驟受眾工作流程。

**歷程**&#x200B;是即時且以事件導向 — 他們會對發生的個別客戶行為做出反應，並在接觸點間協調1:1體驗。 它們最適合用於上線流程、購買後序列和即時觸發訊息。

**行銷活動和歷程**&#x200B;提供您單一授權中的兩項功能。

+++

## 步驟1 — 識別基本選件 {#base-offers}

>[!NOTE]
>
>**正在評估Journey Optimizer？** 在此步驟中，您可以選擇符合參與策略的基本優惠方案。 **已經是客戶嗎？** 以下說明確認您擁有的基本優惠方案及其功能的方式。

提供三個基本選件。 每個角色都符合吸引客戶的不同方式。

| 基本優惠方案 | 最適合 | 核心行為 |
|-----------|---------|--------------|
| **Journey Optimizer — 行銷活動** | 批次、行銷人員規劃的外聯活動 | 以對象為基礎、排程的協調流程。 批次參與和行銷人員計畫外聯的單一步或多步驟行銷活動工作流程。 |
| **Journey Optimizer — 歷程** | 即時客戶參與 | 事件導向，1:1協調流程。 支援即時和已排程的歷程傳遞。 |
| **Journey Optimizer — 行銷活動和歷程** | 同時需要兩者的客戶 | 結合受眾型行銷活動協調與即時歷程協調。 |

>[!IMPORTANT]
>
>**儲存許可權因封裝而異。** 您的總資料量權益取決於您的基本選件： **行銷活動**&#x200B;客戶會收到每個可定址設定檔&#x200B;**15 KB**，而&#x200B;**歷程**&#x200B;和&#x200B;**行銷活動和歷程**&#x200B;客戶會收到每個可定址設定檔&#x200B;**75 KB**。 在選取或確認您的封裝時，將此納入考量。

### 行銷活動vs歷程 — 有何差異？ {#campaigns-vs-journeys}

| | Journey Optimizer — 行銷活動 | Journey Optimizer — 歷程 | Journey Optimizer — 行銷活動和歷程 |
|--|:-----------------------------:|:----------------------------:|:----------------------------------------:|
| 受眾型批次協調 | ✓ | 有限¹ | ✓ |
| 即時事件導向式協調 | — | ✓ | ✓ |
| 異動訊息（電子郵件、推播、簡訊） | ✓ | ✓ | ✓ |
| 可用的管道附加元件 | ✓ | ✓ | ✓ |
| 可用的決策附加元件 | ✓ | ✓ | ✓ |

¹在&#x200B;**Journey Optimizer - Journeys**&#x200B;中，僅歷程使用案例才支援受眾型協調，不支援作為獨立批次行銷活動。

## 步驟2 — 新增您需要的管道 {#channel-addons}

管道未整合到基本選件中。 選取符合您的參與策略的管道附加元件或附加元件套件。

>[!BEGINTABS]

>[!TAB ✉️傳出傳遞]

透過傳出訊息通道觸及對象。

**包含：**&#x200B;電子郵件、推播通知、直接郵件

**典型使用案例：**&#x200B;促銷電子郵件、異動推播警示、實體郵件行銷活動

**支援的介面：**&#x200B;收件匣、行動裝置鎖定畫面/通知匣、郵寄地址

包含新執行個體上IP暖機支援的傳遞性基礎知識。 如需持續監控和顧問領導的支援，有三個&#x200B;**傳遞套件** （Essentials、Enhanced和Plus）可作為個別附加元件提供。 [瞭解傳遞能力](../reports/deliverability.md)

>[!TAB 📱行動裝置]

透過工作階段中和持續性行動體驗，與應用程式使用者互動。

**包含：**&#x200B;應用程式內訊息、推播通知、內容卡、行動介面的程式碼型頻道

**典型使用案例：**&#x200B;上線流程、功能公告、忠誠度提示、即時應用程式內優惠

**支援的介面：**&#x200B;行動應用程式畫面、通知匣、持續性內容槽、透過SDK的自訂應用程式介面

>[!TAB 🌐網頁]

個人化Web體驗，無需部署程式碼。

**包含：**&#x200B;網頁管道（視覺化與非視覺化編輯器）、網頁介面的程式碼型管道

**典型使用案例：**&#x200B;首頁橫幅、登陸頁面個人化、A/B測試、透過API的Headless網頁個人化

**支援的介面：**&#x200B;瀏覽器頁面、單頁應用程式(SPA)、Headless網頁端點

>[!TAB 📦所有頻道]

**所有管道**&#x200B;附加元件可在單次購買中套件組合傳出傳遞+行動+網頁。

**包含：**&#x200B;三個個別附加元件的每個頻道 — 電子郵件、推播通知、直接郵件、應用程式內傳訊、內容卡、Web頻道和程式碼型頻道

**典型使用案例：**&#x200B;跨外寄、行動應用程式和網頁的協調全通路程式 — 例如，在電子郵件後面加上應用程式內訊息和個人化Web橫幅的行銷活動

**支援的介面：**&#x200B;來自傳出傳遞、行動和Web附加元件的所有介面

>[!NOTE]
>
>**All Channels**&#x200B;套件組合通常比個別授權傳出傳遞、行動裝置和網頁更具成本效益。 如果您希望使用這三項，請洽詢Adobe代表，比較套件組合定價和個別附加元件。

>[!ENDTABS]

### 其他管道 {#additional-channels}

某些管道未繫結至單一管道附加元件。 其可用性取決於您的授權組態、合約，以及（針對合作夥伴管道）協力廠商企業帳戶。

**📲簡訊/多媒體簡訊** — 傳送文字和多媒體訊息給行動電話號碼。 每個基本選件都支援異動SMS （即時、事件觸發）；SMS/MMS的更廣泛行銷使用取決於您的授權設定。 [瞭解如何傳送行動訊息](../mobile/get-started-mobile.md)

**💬WhatsApp** — 透過WhatsApp Business傳送訊息。 需要WhatsApp Business帳戶，其可用性取決於您的合約和授權設定。 [瞭解如何使用WhatsApp](../whatsapp/get-started-whatsapp.md)

>[!NOTE]
>
>向您的Adobe代表確認貴組織的SMS/MMS和WhatsApp可用性 — 這些頻道不是標準傳出傳送、行動或網頁附加元件的一部分。

## 步驟3 — 新增進階功能 {#advanced-addons}

### 決策 {#decisioning-addon}

**決策**&#x200B;附加元件可讓您跨任何管道即時定義、管理及傳遞每個設定檔的最佳優惠、動作或體驗。

**它解除鎖定的專案：**
- 使用適用性規則、排名邏輯和限制的即時優惠選擇
- AI支援的排名模型，可自動最佳化優惠方案效能
- 可用於歷程、行銷活動和程式碼型體驗的決策原則

**適用於：**&#x200B;所有三個基本優惠方案，受您的授權合約限制。 [瞭解如何使用決策](../experience-decisioning/gs-experience-decisioning.md) | [瞭解AI模型](../offers/ranking/ai-models.md)

## 常見問題 {#faq}

+++**協調的行銷活動支援哪些管道？**

協調的行銷活動（使用行銷活動協調功能的多步驟對象工作流程）僅支援&#x200B;**電子郵件、簡訊、推播通知和直接郵件**。 協調的行銷活動工作流程不支援Web、應用程式內、程式碼和內容卡通道。

+++

+++**每個設定中是否包含決策？**

沒有。 決策是一種獨特的進階功能附加元件，預設不會包含在任何基本選件中。 請聯絡您的Adobe代表，將Decisioning新增至您的設定。

+++

+++**我聽說過Select、Prime或Ultimate。 這些仍然是目前的封裝模式嗎？**

[!DNL Adobe Journey Optimizer]現在透過圍繞基本優惠方案（行銷活動、歷程、行銷活動和歷程）和附加元件建立的模組化封裝模型提供。 如果您是使用Select、Prime或Ultimate術語的現有客戶，請參閱下方的[舊版封裝](#legacy-packaging)以取得功能參考資料，或聯絡您的Adobe代表確認您目前的權益。

+++

## 舊版封裝 — Select、Prime和Ultimate {#legacy-packaging}

以上各節說明目前的模組化封裝模式。 如果您的合約仍使用&#x200B;**Select**、**Prime**&#x200B;或&#x200B;**Ultimate**&#x200B;術語，請展開下列參考資料，檢視每個舊版套件包含哪些術語。 若要改用模組化模型，請聯絡您的Adobe代表。

+++**顯示舊版封裝參考**

**封裝摘要**

**Select** — 最適合組織開始使用批次和異動訊息：

- 排程的批次行銷活動和交易式訊息
- 核心行銷活動和歷程執行
- 電子郵件、簡訊、推播和自訂動作頻道基礎
- 標準協調護欄

**Prime** — 包含Select中的所有專案，以及即時協調流程和傳入頻道：

- 即時、事件觸發的歷程協調

- 進階對象細分與目標定位

**Ultimate** — 包含Prime中的所有專案，以及決策和進階最佳化：

- 即時優惠決定和個人化
- AI支援的排名和最佳化模型
- 進階報告和實驗功能
