# 開始使用動態內容 {#start-dynamic-content}

>[!BEGINSHADEBOX]

**在此頁面上：**&#x200B;瞭解動態內容如何使用從設定檔屬性、內容事件和對象建立的條件式規則，將您的訊息調整至目標設定檔。

>[!ENDSHADEBOX]

>[!CONTEXTUALHELP]
>id="ajo_conditions_list"
>title="條件"
>abstract="條件式規則可讓您根據輪廓屬性、內容事件或客群在訊息中顯示多個內容變體。"

動態內容可讓您根據&#x200B;**條件式規則**&#x200B;調整訊息內容，這些規則可由輪廓屬性、內容事件或客群組成。 條件式規則是使用個人化編輯器中的視覺化規則產生器來建立，您可以在其中儲存這些規則，以便在歷程及行銷活動中重複使用。

可在電子郵件Designer和個人化編輯器中運用條件式規則，以&#x200B;**建立動態內容**，其將可調整成您訊息中鎖定的設定檔。

>[!TAB 概觀]

**TL；DR**

本頁介紹Journey Optimizer中的動態內容 — 從設定檔屬性、情境事件和對象建立的條件規則如何允許訊息適應它們定位的設定檔。

**個意圖**

* 瞭解什麼是動態內容以及它與靜態訊息內容有何不同
* 識別動態內容的兩個入口點：電子郵件Designer和個人化編輯器
* 導覽至適當的指南，以建立條件式規則或建立動態內容

>[!TAB 字彙]

* **動態內容**：根據條件規則調整訊息內容；根據是否符合定義的條件，不同的內容會顯示給不同的設定檔。 *（產品特定）*
* **條件式規則**：一組條件 — 由設定檔屬性、內容事件或對象所建置 — 可決定設定檔接收的內容變體。 *（產品特定）*
* **視覺化規則產生器**：個人化編輯器中的互動式畫布，用來建立和儲存條件式規則。 *（產品特定）*

>[!TAB 術語]

* **正式名稱：**&#x200B;動態內容 — 變體：條件式內容，個人化內容
* **同義字：** &quot;conditional rules&quot; = &quot;conditions&quot; （如UI中標示）
* **請勿混淆：**&#x200B;動態內容（根據條件規則調整每個設定檔的內容）≠靜態內容（不論條件為何，都會向所有收件者顯示相同的內容）

>[!TAB 常見問題集]

**問：條件規則可以使用哪些型別的條件？**

可從設定檔屬性、情境式事件和對象成員資格建立條件式規則。

**問：條件式規則可在何處用來建立動態內容？**

在電子郵件Designer （建立元件變體）和個人化編輯器中（使用條件包裝運算式區塊）。 兩個入口點都會包含在連結的頁面中。

**問：條件式規則是否可以在歷程與行銷活動中重複使用？**

有。 可儲存條件式規則，以便在歷程及行銷活動中進一步重複使用。
