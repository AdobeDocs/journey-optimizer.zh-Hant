# 決策資料集{#decisions-dataset}

每次修改選件時，就會更新自動產生的決策資料集。

![](../../assets/dataset-activities.png)

資料集中最近成功的批次會顯示在右側。 資料集的架構階層檢視會顯示在左窗格上。

>[!NOTE]
>
>瞭解如何在[本節](../export-catalog/access-dataset.md)中存取選件程式庫每個物件的匯出資料集。

決策（先前稱為選件決策）用於控制決策過程。 它會指定套用至總庫存的篩選條件，以依主題／類別縮小選件，將庫存縮小至技術上符合選件預留空間的選件的位置，並指定在組合限制不符合所有可用個人化選件的資格時的備援選項。

以下是&#x200B;**[!UICONTROL Decision Object Repository - Decisions]**&#x200B;資料集（先前稱為Decision Object Repository - Activities）中可使用的所有欄位清單。

## 識別碼

記錄的唯一識別碼。

類型: 字串

## _體驗

### 決策

#### 准則

定義一組決策標準，其中每個標準都包含一組約束。

類型：陣列

* **說明**

   標準說明。 它用於傳達人類可讀意圖，以瞭解如何構建或為何構建此標準以及它如何影響決策。

   類型: 字串

* **選項選擇**

   選項選擇定義了選項在此上下文中的有效性／適用性。

   類型：物件

   * **說明**

      選項選擇說明。 它用於傳達人類可讀意圖，瞭解如何或為何構建此選項選擇和／或哪個選項匹配。

      類型: 字串

   * **選項篩選**

      對基於標籤的篩選器的參考，該篩選器使用其附加的標籤來比對庫存的選項。 該值是所引用決策規則的URI(@id)。 請參閱架構https://ns.adobe.com/experience/decisioning/filter。

      類型: 字串

   * **描述檔約束類型**

      確定當前是否設定了任何約束以及約束的表達方式。 可能是透過篩選查詢或透過一或多個區段成員資格。

      類型: 字串

   * **選項清單**

      直接指定選項而不評估篩選器查詢的清單。 可以指定選項清單或選項篩選規則。

      類型：陣列

      <!--Missing title under Option List? Desc = An identifier of an decision option entity. The value value refers to an `@id` property of a decision option.-->

* **位置**

   放置約束表示此准則僅適用於列出的位置。 只有當目標位置位於`xdm:placements`清單中時，才會考慮選項選擇。 否則，將跳過整個決策標準。 當省略或空白「xdm:lactions」清單時，任何定位的位置都會考慮標準。 此處列出的位置會為選項選擇加上隱含標準。 要考慮的選項必須具有目標放置的表示法。

   類型：陣列

   * **位置識別碼**

   放置圖元的參照。 該值是所引用位置的URI(@id)。 請參閱架構https://ns.adobe.com/experience/decisioning/placement。

   類型: 字串

* **描述檔約束**

   描述檔約束會決定在此情況下，目前某個選項選擇是否符合此描述檔識別的資格。 如果描述檔約束不需要考慮每個選項的值，即選項選擇中的選項不變，則評估為&#39;false&#39;的描述檔約束會取消整個選項選擇。 另一方面，系統會針對選項選擇的每個限定選項評估以選項作為參數的配置檔案約束規則。

   類型：物件

   * **說明**

      描述檔約束說明。 它用於傳達人類可讀意圖，瞭解如何或為何構建此配置檔案約束和／或該約束將包含或排除哪個選項。

      類型: 字串

   * **資格規則**

      對決策規則的引用，該決策規則對給定的配置檔案和／或其他給定的上下文XDM對象評估為true或false。 此規則用於決定選項是否符合指定描述檔的資格。 該值是所引用決策規則的URI(@id)。 請參閱架構https://ns.adobe.com/experience/decisioning/rule。

      類型: 字串

   * **描述檔約束類型**

      確定當前是否設定了任何約束以及約束的表達方式。 可能是透過規則或透過一或多個區段成員資格。

      類型: 字串

      可能的值：&quot;none&quot;、&quot;agilibilityRule&quot;、&quot;anySegments&quot;、&quot;allSegments&quot;、&quot;rules&quot;

      預設值：&quot;none&quot;

   * **區段識別碼**

      區段的識別碼

      字串：陣列

      * **識別碼**

         相關名稱空間中區段的識別。

         類型: 字串

      * **命名空間**

         與`xid`屬性關聯的命名空間。

         類型：物件

         * **程式碼**

            代碼是命名空間的人類可讀標識符，可用於請求用於身份圖處理的技術命名空間ID。

            類型: 字串
      * **體驗識別碼**

         當存在時，此值表示跨名稱空間標識符，該標識符在所有名稱空間中所有命名空間範圍內的標識符中都是唯一的。

         類型: 字串


* **排名詳細資訊**

   排名（優先順序）。 定義在決策標準的上下文下如何確定\&quot;最佳選項\&quot;。 在符合描述檔約束的所有選取選項中，排名將決定要建議的頂端（或前N個）選項。

   類型：物件

   * **訂單評估**

      評估一個或多個決策選項的相對順序。 對於具有較低序數值的任何選項，都會選擇具有較高序數值的選項。 該方法所確定的值可以排序，但是它們之間的距離不能測量，且不能求和或求積。 中值和模式是唯一可用於順序資料的中心趨勢度量。

      類型：物件

      * **計分函式**

         計算此決策選項的數值分數的函式參考。 然後，將依該分數排序（排名）決策選項。 此屬性的值是一次使用on選項調用的函式的URI(@id)。 請參閱架構https://ns.adobe.com/experience/decisioning/function。

         類型: 字串

      * **訂單評估類型**

         指定使用哪種訂單評估機制、決策選項的靜態優先順序、計算每個選項的數值的評分函式或接收用於排序的清單的排名策略。

         類型: 字串

      * **排名策略**

         對策略的引用，該策略對決策選項清單進行排名。 決策選項將在有序清單中返回。 此屬性的值是一次使用on選項調用的函式的URI(@id)。 請參閱架構https://ns.adobe.com/experience/decisioning/rankingStrategy。

         類型: 字串
   * **優先順序**

      單一決策選項相對於所有其他選項的優先順序。 使用此屬性，不提供訂單函式的選項會排定優先順序。 優先順序值較高的選項會在任何優先順序較低的選項之前選取。 如果兩個或兩個以上符合資格的期權具有最高的優先順序值，則會以一致的隨機方式選擇一個期權，用於決策建議。

      類型：整數

      最小值：0

      預設值：0


#### 活動結束日期和時間

決定結束日期和結束時間。 屬性在http://schema.org/Action上定義schema.org&#39;endTime&#39;屬性的語義。

類型: 字串

#### 備援選項

在此決策中進行決策時，對備援選項的參考不會限定任何常規選項（這通常發生在套用硬約束時）。 該值是引用的備援選項的URI(@id)。

類型: 字串

#### 活動名稱

顯示在各種使用者介面中的決策名稱。

類型: 字串

#### 活動開始日期和時間

決定開始日期和結束時間。 屬性在http://schema.org/Action上定義schema.org&#39;s startTime&#39;屬性的語義。

類型: 字串

類型: 字串
