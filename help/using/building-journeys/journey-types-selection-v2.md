# 歷程型別：選擇正確的型別 {#journey-types-selection}

>[!BEGINSHADEBOX]

**在此頁面上：**&#x200B;瞭解四種AJO歷程型別 — 單一事件、讀取對象、對象資格和業務事件 — 並找出適合您使用案例的型別。

>[!ENDSHADEBOX]

[!DNL Adobe Journey Optimizer]支援四種歷程型別，每種都針對不同型別的觸發程式和業務案例而設計。 瞭解差異有助於您從一開始就建立正確的體驗。

## 四種歷程型別 {#journey-types}

>[!BEGINTABS]

>[!TAB 單一事件歷程]

**何時使用：**&#x200B;即時事件觸發的體驗

**單一事件歷程**&#x200B;會在特定動作（購買、應用程式登入、表單提交）發生時個別觸發。 設定檔一次會即時輸入一個，因此非常適合用於即時行為導向的回應。

**最適合：**&#x200B;購物車放棄復原、新會員上線、有人訂閱時的歡迎電子郵件，以及登入後個人化。

➡️ [瞭解事件](../event/about-events.md) | [傳送訊息給訂閱者使用案例](message-to-subscribers-uc.md) | [建立您的第一個歷程](journey-gs.md)

>[!TAB 讀取對象歷程]

**何時使用：**&#x200B;已排程行銷活動給對象區段

**讀取對象歷程**&#x200B;從[!DNL Adobe Experience Platform]個對象開始，並同時以批次傳送訊息給所有設定檔。 此歷程型別適用於已排程的大型通訊。 使用週期性歷程的&#x200B;**增量讀取**&#x200B;選項，僅處理自上次執行以來加入對象的設定檔，而不是每次都重新處理完整對象。

**最適合：**&#x200B;每月電子報、目標區段的促銷活動、產品公告、週期性重新參與系列和季節性行銷活動。

➡️ [瞭解閱讀對象](read-audience.md) | [開始使用對象](../audience/about-audiences.md) | [建立您的第一個歷程](journey-gs.md)

>[!TAB 對象資格歷程]

**何時使用：**&#x200B;對象成員資格變更的即時回應

當設定檔符合（或退出）特定對象的資格時，**對象資格歷程**&#x200B;會觸發。 設定檔在符合條件時個別輸入，可在客戶行為變更時立即參與。 使用&#x200B;**串流評估**&#x200B;對象 — 這些是此活動唯一支援的對象型別。

>[!CAUTION]
>
>自&#x200B;**2026年8月**&#x200B;起，無法在對象資格節點中使用批次對象的歷程無法發佈。 [瞭解如何移轉您的歷程](aq-batch-audiences-migration.md)

**最適合：** VIP層級升級通知、首次購買慶祝訊息、流失風險警示以及忠誠度生命週期階段轉換。

➡️ [瞭解對象資格](audience-qualification-events.md) | [建立對象](../audience/creating-a-segment-definition.md) | [建立您的第一個歷程](journey-gs.md)

>[!TAB 商務活動歷程]

**何時使用：**&#x200B;影響多個客戶的業務狀況

**業務事件歷程**&#x200B;是由同時影響多個設定檔的業務層級事件（庫存更新、價格變更）觸發。 在內部，業務事件觸發程式一律會依循擷取相關設定檔的讀取對象步驟，因此設定檔專案會依循讀取對象輸送量規則，而非單一事件輸送量。

**完美搭配：**&#x200B;向感興趣的客戶發出低庫存警示、快閃銷售公告、降價通知以及產品補貨警示。

➡️ [瞭解業務活動](../event/about-creating-business.md) | [專案管理](entry-management.md) | [建立您的第一個歷程](journey-gs.md)

>[!ENDTABS]

## 您應該使用哪種型別？ {#decision-guide}

答案通常歸結為一個問題：*什麼開始了歷程？*

如果&#x200B;**客戶做了特定事情** — 放棄購物車、註冊、購買 — 使用&#x200B;**單一事件歷程**。 它會在動作發生時立即引發，一次一個設定檔。

如果您想要&#x200B;**依排程** （每月電子報、季節性行銷活動、週期性重新參與系列）觸及對象，請使用&#x200B;**讀取對象歷程**。 您可以定義對象和時間；AJO會一次處理每個人。

如果您想要在客戶達到里程碑時&#x200B;**回應** — 加入忠誠度等級、達到流失風險臨界值、完成首次購買 — 請使用&#x200B;**對象資格歷程**。 當串流受眾成員資格變更時，就會立即觸發，而非依固定排程觸發。

如果您的企業中有某些事情變更了&#x200B;****，而這個變更同時影響了多位客戶 — 庫存水準下降、價格變更、銷售開始 — 請使用&#x200B;**業務事件歷程**。

>[!TIP]
>
>**不確定從何處開始？** 對於行為觸發的體驗，大部分團隊以&#x200B;**單一事件**&#x200B;開始，對於行銷活動，以&#x200B;**讀取對象**&#x200B;開始。 這兩者涵蓋大部分的使用案例。

| 您的目標 | 建議的歷程型別 | 原因 |
|-----------|--------------------------|-----|
| 復原捨棄的購物車 | 單一事件 | 對個人行為立即回應 |
| 傳送每月電子報給訂閱者 | 讀取對象 | 排定的批次通訊 |
| 當客戶達到VIP狀態時通知客戶 | 客群鑑定 | 對串流受眾進入的即時回應 |
| 提醒客戶留意已觀看專案的庫存不足 | 業務事件 | 業務狀況影響多個客戶 |
| 歡迎新應用程式使用者 | 單一事件或對象資格 | 註冊事件（單一事件）或進入新使用者串流對象（對象資格） |
| 重新與非作用中客戶互動（週期性、已排程） | 讀取對象 | 針對非活動對象重複執行批次 |
| 目標區段的季節性促銷 | 讀取對象 | 已排程行銷活動給對象 |
| 快閃銷售公告 | 業務事件 | 業務決策影響多個客戶 |
| 客戶達到金級忠誠度時立即回應 | 客群鑑定 | 串流對象，即時個人登入 |

## 功能可用性參考 {#feature-compatibility}

所有歷程型別都支援完整的AJO頻道集（電子郵件、推播、簡訊、應用程式內、網頁、內容卡）、核心協調活動（等待、條件、自訂動作）、測試模式、試執行以及傳送時間最佳化。 下表只顯示不同型別的功能。

>[!NOTE]
>
>跳轉活動限制：以讀取對象或對象資格活動開始的歷程不能包含跳轉活動，且不能成為其他歷程中跳轉活動的目標。
>
>讀取對象活動，因為歷程專案僅適用於&#x200B;**讀取對象**&#x200B;和&#x200B;**商業事件**&#x200B;歷程 — 無法將其新增至單一事件或對象資格專案歷程。

| 功能 | 單一事件 | 讀取對象 | 客群鑑定 | 業務事件 |
|-----------|:-------:|:-------------:|:----------------------:|:--------------:|
| **專案** | | | | |
| 事件觸發的專案 | ✅ | ❌ | ❌ | ✅ （業務事件會觸發歷程；設定檔會透過內部讀取對象步驟進入） |
| 排定的專案 | ❌ | ✅ | ❌ | ❌ |
| 受眾型登入 | ❌ | ✅ （批次） | ✅ （僅限串流） | ❌ |
| **協調流程** | | | | |
| 讀取對象活動（歷程專案） | ❌ | ✅ | ❌ | ✅ （業務事件後的自動步驟） |
| 跳轉活動 | ✅ | ❌ | ❌ | ✅ |
| **設定檔管理** | | | | |
| 設定檔重新進入 | ✅可設定 | ❌預設每次執行一次（[週期時強制重新進入](read-audience.md#schedule)可用） | ✅可設定（已在歷程中的設定檔無法重新輸入相同版本） | 每個事件✅ |
| **最佳化** | | | | |
| 路徑實驗（A/B測試） | ✅ | ✅ | ✅ | ❌ |
| **進階** | | | | |
| 增量讀取 | ❌ | ✅ | ❌ | ❌ |
| 最大輸送量 | 5,000 TPS （共用組織層級與對象資格） | 每個沙箱20,000 TPS | 5,000 TPS （共用組織層級與單一事件） | 業務活動：5,000 TPS；讀取對象步驟：20,000 TPS |

**圖例：** ✅ =支援 | ❌ =不支援

**意圖：**

* 使用決定表格為特定業務使用案例選擇正確的歷程型別
* 使用詳細的功能相容性矩陣，並排比較歷程型別
* 瞭解何時將讀取對象歷程用於已排程的批次通訊
* 瞭解何時使用單一事件歷程提供即時、事件觸發的體驗
* 瞭解何時將對象資格歷程用於即時狀態變更回應
* 瞭解何時將業務事件歷程用於業務條件導向的通訊
* 瞭解規劃大量部署時每個歷程型別的輸送量限制

**字彙表：**

* **單一事件歷程**：由特定個別客戶動作（例如，購買、登入）觸發的歷程，其中設定檔一次會即時輸入一個。 *（產品特定）*
* **讀取對象歷程**：以Adobe Experience Platform對象開始的歷程，並依排程同時批次傳送訊息至所有設定檔。 *（產品特定）*
* **對象資格歷程**：設定檔符合或退出特定對象區段時觸發的歷程。 需要串流評估的對象才能呈現即時進入行為。 *（產品特定）*
* **業務事件歷程**：由業務層級事件（例如，庫存更新、價格變更）觸發的歷程，同時影響多個設定檔；一律與設定檔擷取的內部讀取對象步驟配對。 *（產品特定）*
* **增量讀取**：讀取對象功能，僅處理自上次執行以來加入對象的設定檔，而不是每次都處理完整的對象。 僅供讀取對象歷程使用。 *（產品特定）*
* **串流對象**： Adobe Experience Platform對象會持續即時評估，而非依排程評估的批次對象（例如每日）。 對象資格歷程需要此專案，才能取得即時進入行為。 *（產品特定）*

**護欄：**

* 增量讀取僅適用於讀取對象歷程，不適用於單一事件、對象資格或業務事件歷程
* 業務事件歷程不支援路徑實驗（A/B測試）
* 依預設，讀取對象歷程中的設定檔重新進入僅限於每次執行一次；在排程執行時重複使用強制重新進入，以允許設定檔在下次執行時重新進入
* 讀取對象活動僅在讀取對象和業務事件歷程中作為歷程專案提供 — 而不是在單一事件或對象資格專案歷程中
* 對象資格和讀取對象歷程不能包含跳轉活動，且不能是另一個歷程中跳轉活動的目標
* 對象資格歷程需要串流評估的對象。 從2026年8月開始，批次評估的對象不能用於對象資格節點 — 請參閱[移轉指南](aq-batch-audiences-migration.md)
* 單一事件和受眾資格歷程在組織層級都有5,000個TPS輸送量限制；讀取受眾歷程支援每個沙箱最多20,000個TPS
* 歷程中已存在的設定檔無法重新進入該歷程的相同版本，無論重新進入設定為何

**術語：**

* 正式名稱：單一事件歷程 — 變體：事件觸發的歷程、單一歷程
* 正式名稱：讀取對象歷程 — 變體：批次歷程
* 正式名稱：對象資格歷程 — 變體：對象資格事件歷程
* 正式名稱：業務事件歷程 — 變體：業務事件觸發的歷程
* 請勿混淆：「讀取對象歷程」≠「對象資格歷程」 — 讀取對象會依排程批次處理所有對象成員；對象資格會即時回應個別成員資格變更（串流對象僅供即時進入）
* 請勿混淆：「單一事件歷程」≠「業務事件歷程」 — 單一是由影響一個設定檔的客戶動作觸發；業務事件是由業務條件觸發，並透過內部讀取對象步驟擷取多個設定檔

**常見問題集：**

* **問：我應使用哪個歷程型別來製作每月電子報？**  — 使用讀取對象歷程；其設計用於同時與對象區段中的所有設定檔進行排程批次通訊。
* **問：我應該使用哪個歷程型別來復原放棄的購物車？**  — 使用單一事件歷程；它會在發生放棄事件時立即觸發，並即時回應個人的行為。
* **問：我可以在商務活動歷程中執行A/B路徑實驗嗎？**  — 否；業務事件歷程不支援路徑實驗。
* **問：單一事件歷程與對象資格歷程之間有何差異？**  — 單一事件歷程由特定客戶動作（例如購買）觸發；當設定檔根據串流條件評估進入或退出對象區段時，就會觸發對象資格歷程。
* **問：哪些歷程型別支援增量讀取？**  — 只有讀取對象歷程支援增量讀取，其他三種歷程型別不支援。
* **問：我可以將「讀取對象」活動新增至單一事件歷程嗎？**  — 否；「讀取對象」活動僅在讀取對象和商務事件歷程中作為歷程專案提供。
* **問：我可以在讀取對象歷程中使用跳轉活動嗎？**  — 否；以讀取對象或對象資格活動開始的歷程不能包含跳轉活動，且不能作為來自另一個歷程的跳轉的目標。
* **問：我是否可歡迎有對象資格歷程的新應用程式使用者？**  — 可以，如果專案是由串流對象驅動（例如，當設定檔加入新的使用者區段時）；註冊單一事件歷程也是常見模式。
* **問：我的對象資格歷程未即時觸發。 為什麼？**  — 對象資格歷程需要串流評估的對象。 已棄用批次評估的對象，並將從2026年8月起封鎖。 [請參閱移轉指南](aq-batch-audiences-migration.md)
* **問：單一事件和讀取對象歷程之間的輸送量差異為何？**  — 單一事件歷程與組織層級的對象資格歷程共用5,000個TPS限制。 讀取對象歷程支援每個沙箱最多20,000個TPS，使其更適合大規模批次行銷活動。
