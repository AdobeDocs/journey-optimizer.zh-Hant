# 開始使用歷程模擬 {#simulate-journey-gs}

>[!IMPORTANT]
>
>您至少需要下列其中一個許可權才能存取&#x200B;**[!UICONTROL 模擬]**&#x200B;功能： **模擬歷程**、**發佈歷程**&#x200B;或&#x200B;**核准並發佈歷程**。 [了解更多](../administration/permissions.md)

除了&#x200B;**草稿**、**測試模式**&#x200B;和&#x200B;**即時**&#x200B;之外，您還可以將歷程設定為&#x200B;**[!UICONTROL 模擬]**。 在模擬中，您使用&#x200B;**個模擬的使用者進行測試**：您新增的臨時設定檔樣實體，而不使用Adobe Experience Platform中的持續測試設定檔。

Adobe Journey Optimizer提供兩種方式來測試和驗證您的歷程：

* **[模擬](simulate-journey.md#test-users)**：使用&#x200B;**[!UICONTROL 模擬]**&#x200B;歷程功能，並模擬沒有在Adobe Experience Platform中預先建立設定檔的使用者，同時支援AI支援和手動建立的使用者。

* **[測試模式](testing-the-journey.md)**：使用在Adobe Experience Platform中標示為測試設定檔的持續設定檔，可跨工作階段重複使用。 當您需要一致、預先定義的資料時，請選擇此方法。 [瞭解如何建立測試設定檔](../audience/creating-test-profiles.md)。

## 依歷程型別模擬 {#by-journey-type}

**[!UICONTROL 模擬]**&#x200B;面板只會顯示您歷程所需的步驟。 這取決於設定檔進入歷程的方式。 根據這些因素，Adobe Journey Optimizer會呈現不同的模擬體驗。 展開下方的每個型別，檢視執行的差異以及您使用的面板。

如需詳細資訊，請參閱[模擬您的歷程](simulate-journey.md)。

+++ 具有讀取對象的批次歷程


歷程是由&#x200B;**[!UICONTROL 讀取對象]**觸發，而且畫布沒有單一事件活動。模擬期間不會觸發對象母體。只有模擬的使用者進入歷程。
為模擬選取的模擬使用者會出現在**測試使用者**&#x200B;區段中：

![僅讀取對象之批次歷程的模擬面板](assets/simulate-batch.png)

+++

+++ 具有讀取對象和單一事件的批次歷程

包括沿路徑的一或多個單一事件的區段觸發歷程。您先會觸發模擬使用者輸入模擬，然後為在事件節點等待的使用者觸發事件。
針對模擬和已設定事件選取的模擬使用者將分別顯示在測試使用者和測試事件區段中。在模擬使用者進入歷程之前，不會顯示測試事件區段。

![僅讀取對象之批次歷程的模擬面板](assets/simulate-batch-2.png)

+++

+++ 單一歷程

歷程從單一事件開始，而不是讀取對象。直到為其觸發該開始事件後，模擬使用者才會進入歷程。
為模擬選取的模擬使用者與設定的事件將分別顯示在**測試使用者**&#x200B;與&#x200B;**測試事件**&#x200B;區段中。**測試使用者**&#x200B;區段不包含將模擬使用者觸發歷程的動作。您從&#x200B;**測試事件**&#x200B;觸發專案。

![僅讀取對象之批次歷程的模擬面板](assets/simulate-batch-3.png)

+++

## 啟動模擬 {#launch}

將歷程切換為&#x200B;**[!UICONTROL 模擬]**&#x200B;以與模擬的使用者測試。 在[模擬您的歷程](simulate-journey.md)中會詳細說明逐步工作。

1. 在您的歷程中，按一下&#x200B;**[!UICONTROL 模擬]**&#x200B;並選擇&#x200B;**[!UICONTROL 模擬]**。

   歷程介面中的![測試模式按鈕](assets/test-mode-simulated.png)

1. 等候啟用完成。 當歷程切換至&#x200B;**[!UICONTROL 模擬]**&#x200B;時，面板中的控制項會停用，並在啟動完成後自動重新啟用。

## 限制 {#limitations}

在這個版本中，**[!UICONTROL 模擬]**&#x200B;可能不支援&#x200B;**[!UICONTROL 測試模式]**&#x200B;或即時歷程支援的所有活動、管道或整合，而且行為可能會隨著功能成熟而改變。 請針對支援的工作流程使用本文。

請參閱下方的下拉式清單，以進一步瞭解模擬限制。

+++ 節點層級限制

某些節點會阻止&#x200B;**[!UICONTROL 模擬]**&#x200B;啟動。 其他則使用下述行為在模擬中執行。 當您在模擬之前必須移除或變更節點時，請先更新歷程。

| 受限制的節點 | 附註 |
| --- | --- |
| 業務事件 | 您無法在&#x200B;**[!UICONTROL 模擬]**&#x200B;中執行以業務事件開始的歷程。 |
| 補充ID （多次重新進入） | 啟用多重重新進入時，**[!UICONTROL 模擬]**&#x200B;不會啟動，而且同一個模擬使用者可能同時具有多個作用中的執行個體。 |
| 內容決定節點 | 在您模擬歷程之前，請移除或變更此活動。 |
| 資料集查詢 | **[!UICONTROL 模擬]**&#x200B;不支援依索引鍵查詢客戶資料集。 在執行模擬之前，請移除或變更此活動。 |
| **[!UICONTROL 最佳化]**&#x200B;活動 | 不支援&#x200B;**[!UICONTROL 實驗]**&#x200B;和&#x200B;**[!UICONTROL 目標規則]**。 在您模擬之前移除或變更節點。<br><br>其他&#x200B;**[!UICONTROL 最佳化]**&#x200B;方法的行為如下：<br><br>**[!UICONTROL 百分比分割&#x200B;]**： Journey Agent會為每個分支建立一個模擬使用者，而不是根據分支百分比。 在執行階段，即時評估會挑選分支，而且可能與產生的路徑不同。 您無法模擬分支選擇。 若要引導使用者，請仰賴畫布上的分支順序。 總是選擇頂端分支。<br><br>**[!UICONTROL 時間條件]**：條件會在執行階段套用，如同在即時歷程中一樣。 例如，從8:00到20:00的視窗只允許使用者在該視窗內執行模擬時通過。 您無法模擬執行時間。 設定符合目前測試時間的條件。<br><br>**[!UICONTROL 日期條件&#x200B;]**：條件會在執行階段套用，如同在即時歷程中一樣。 例如，2026年6月8日這個日期只允許使用者在該日期執行模擬時通過。 您不能模擬執行日期。 將條件設定為您測試時的目前日期。<br><br>**[!UICONTROL 設定檔上限]**：模擬期間未強制執行上限。 Journey Agent會為每個分支建立一個模擬使用者。 您無法模擬分支選擇。 若要引導使用者，請仰賴畫布上的分支順序。 總是選擇頂端分支。 |
| 逾時和錯誤分支 | Journey Agent不會針對活動逾時或錯誤分支產生使用者。 只有在模擬期間發生實際的逾時或錯誤時，使用者才會輸入這些路徑。 |
| 逾時分支（事件活動） | 已建立模擬使用者，但在&#x200B;**[!UICONTROL 手動模擬]**&#x200B;中，Journey Agent不會決定誰進入事件逾時分支。 透過傳送或不傳送事件來控制路徑。 例如，若要測試逾時分支，請等候逾時設定，並且不要傳送事件。 **[!UICONTROL 快速模擬]**&#x200B;可以自動傳送或保留事件以涵蓋逾時分支。 |
| 反應事件 | 反應事件會在模擬中執行，但動作必須在現實生活中發生。 例如，電子郵件&#x200B;**開啟**&#x200B;回應需要開啟校樣訊息。 您無法在模擬UI中模擬回應。 |
| 外部資料來源 | 呼叫在模擬期間的執行方式與即時歷程中相同。 下游活動可以使用回應，但您無法加以模擬。 當回應值摘要&#x200B;**[!UICONTROL 最佳化]**&#x200B;活動時，Journey Agent無法發明該輸出。 它只會產生呼叫的輸入。 例如，如果呼叫取用設定檔城市並傳回天氣，則「代理程式」會為模擬使用者設定城市，而即時呼叫會傳回天氣。 |
| 自訂動作 | 行為符合外部資料來源。 傳出呼叫會實際執行。 Journey Agent會填入輸入。 輸出來自即時回應。 您無法模擬回應。 |
| 外部對象屬性擴充 | 套用此驗證時，使用來自外部對象來源的個人化屬性的歷程不會在&#x200B;**[!UICONTROL 模擬]**&#x200B;中開始。 |

+++

+++ 功能限制

**[!UICONTROL 模擬]**&#x200B;不支援下列功能。

| 功能 | 附註 |
| --- | --- |
| 退出條件 | 當您執行&#x200B;**[!UICONTROL 模擬]**&#x200B;時，不會套用退出條件。 |
| 動作內的[!DNL Adobe Journey Optimizer]決策，例如Adobe Journey Optimizer決策的電子郵件內容 | 不產生使用[!DNL Adobe Journey Optimizer]決策之內容的動作校樣。 |
| 模擬自訂動作回應 | [!UICONTROL 自訂動作]預設會執行實際傳出呼叫。 正在模擬回應，因此不支援執行任何外部呼叫。 |
| 同意原則評估 | 在模擬使用者層級上無法模擬同意，而且在模擬期間不會評估同意原則。 |
| 歷程上限和仲裁 | 模擬期間未評估或執行。 |
| 頻率限定（依頻道或通訊型別） | 模擬期間未評估或執行。 |
| 選擇退出管理、隱藏和允許清單 | 模擬期間未評估或套用。 |
| 頻道設定中的動態子網域和動態屬性 | 不支援。 |
| 傳送時間最佳化(STO) | 模擬期間未評估或套用。 |
| 沙箱工具（跨沙箱複製模擬使用者） | 不支援。 |
| 在歷程中傳送的波次 | 不支援。 |
| 無訊息時間 | 模擬期間未評估或套用。 |
| Privacy service | 模擬的使用者不是符合GDPR的永久性設定檔。 請勿在模擬的使用者中包含真實的客戶資料。 |

| 護欄 | 限制 | 附註 |
| --- | --- | --- |
| 可以選取和觸發一個批次的最大模擬使用者數（批次歷程、事件觸發的流程和對象資格流程） | 20 | 每個&#x200B;**[!UICONTROL 傳送全部]**&#x200B;或&#x200B;**[!UICONTROL 觸發選取的事件]**&#x200B;都會計入，而非整個歷程的累積上限。 |
| 每一代請求的最大模擬使用者數 | 50 | 透過&#x200B;**[!UICONTROL 快速模擬]**&#x200B;或&#x200B;**[!UICONTROL 在**[!UICONTROL &#x200B;手動模擬&#x200B;]**中使用AI]**&#x200B;產生，Journey Agent在一個要求中產生的模擬使用者上限。 如果歷程具有超過&#x200B;**50**&#x200B;個路徑，Journey Agent會隨機選取路徑以產生這&#x200B;**50**&#x200B;個模擬使用者。 |
| 在單一模擬回合中測試的不重複模擬使用者上限 | 100 | 在一個執行區塊中聯絡&#x200B;**100**&#x200B;個不重複使用者&#x200B;**[!UICONTROL 為新的模擬使用者選取模擬使用者]**。 如果您位於&#x200B;**90**，在相同區塊之前最多可以新增&#x200B;**10**。 |
| 可以在一個沙箱中同時在&#x200B;**[!UICONTROL 模擬]**&#x200B;中執行的最大歷程數 | 20 | Cap一次由該沙箱中的每個&#x200B;**[!UICONTROL 模擬]**&#x200B;歷程共用。 |
| 一個沙箱中最大活動模擬使用者數 | 2,000 | 一次可存在於沙箱中的最大模擬使用者數。 Adobe可能會根據客戶意見反應調整此限制。 |
| 事件預填（僅限瀏覽器） | — | 您只能在瀏覽器型模擬UI中預先填寫事件裝載欄位。 預先填入的值只會保留在瀏覽器中，不會同步至其他瀏覽器、裝置或工作階段，因此您可能會在每個測試位置看到不同的預先填入資料。 |
