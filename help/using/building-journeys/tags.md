# 管理歷程中的標籤 {#journey_tags}

>[!BEGINSHADEBOX]

**在此頁面上：**&#x200B;瞭解如何使用標籤和標籤類別來組織歷程，以便比命名慣例更輕鬆地分類、篩選和尋找您的歷程。

>[!ENDSHADEBOX]

作為Journey Optimizer的從業人員，您可以使用標籤來組織您的歷程。 標籤是一種快速、簡便的物件分類方法，用於改進搜尋。

## 標籤與命名慣例的比較 {#tags-vs-naming}

團隊通常依賴複雜的命名慣例將中繼資料直接儲存在歷程名稱中 — 例如： *生命週期行銷 — 教育 — 客戶入門V2 — 應用程式教育 — 2025年第3季*。 雖然是出於善意，但這種方法有一個關鍵弱點：隨著工作跨團隊成員調整，慣例很少會一致地應用，而歷程清單會變得難以導覽。

Journey Optimizer中的&#x200B;**標籤類別**&#x200B;可提供更好的替代方案。 您不需要在名稱中編碼中繼資料，而是將分類的標籤附加至每個歷程（例如團隊、目標、階段、季度），並使用篩選器來尋找它們。 接著，歷程名稱就可以聚焦於真正重要的專案：所驅動的客戶里程碑。

標籤類別優於命名慣例的優點：

* **一致性** — 標籤是從控制清單中選取的，不是自由輸入。
* **可篩選性** — 任何標籤值的組合都可以用來立即分割歷程清單。
* **清晰度** — 歷程名稱保持簡短且以里程碑為中心。
* **擴充性** — 新增中繼資料維度表示建立新的標籤類別，而非重寫命名慣例。

如需建議的設定工作流程，請參閱下方的[設定歷程管理的標籤類別](#tags-setup)。

## 將標籤新增至歷程

歷程屬性中的&#x200B;**標籤**&#x200B;欄位可讓您定義歷程的標籤。 您可選取現有標籤或建立新標籤。 開始輸入所需標籤的名稱，然後從清單選取該名稱。 如果無法使用，請按一下&#x200B;**建立**&#x200B;以建立新的歷程並將其新增至您的歷程。 您可視需要定義任意數量的標籤。

歷程屬性中的![標籤面板，用於分類和組織](assets/tags1.png)

定義的標籤清單會顯示在&#x200B;**標籤**&#x200B;欄位下方。

>[!NOTE]
>
> 標籤不區分大小寫
> 
> 如果您複製或建立新版本的歷程，則會保留標籤。

## 依標籤篩選

歷程清單會顯示專用欄，讓您輕鬆地將您的標籤視覺化。

篩選器也可用於僅顯示具有特定標籤的歷程。

![標籤選取下拉式清單，其中包含歷程分類的可用標籤](assets/tags2.png)

您可以從任何型別的歷程（即時、草稿等）新增或移除標籤。 按一下歷程旁的&#x200B;**更多動作**&#x200B;圖示，然後選取&#x200B;**編輯標籤**。

![依標籤篩選的歷程清單，顯示已分類的歷程](assets/tags3.png)

## 管理標籤

管理員可利用&#x200B;**管理**&#x200B;下方的&#x200B;**標籤**&#x200B;選單來刪除標籤，並依類別進行管理。 請參閱此[檔案](https://experienceleague.adobe.com/docs/experience-platform/administrative-tags/overview.html?lang=zh-Hant)。

>[!NOTE]
>
> 歷程中定義的標籤會新增至內建的「未分類」類別。

## 設定歷程管理的標籤類別 {#tags-setup}

請依照下列步驟，在您的團隊中以標籤式方法取代複雜的命名慣例。

**步驟1 — 建立標籤類別（管理員）**

在&#x200B;**[!UICONTROL 管理]** > **[!UICONTROL 標籤]**&#x200B;中，為您的團隊目前在歷程名稱中編碼的每個中繼資料屬性建立一個類別 — 例如： *團隊*、*行銷目標*、*行銷活動*、*階段*、*季*。

**步驟2 — 將標籤值(Admin)填入每個類別**

在每個類別中，建立代表所有可能值的標籤。 例如，*階段*&#x200B;類別可能包含： *感知*、*上線*、*保留*、*回覆*。

**步驟3 — 建立歷程（從業者）時套用標籤**

每次建立新歷程時，請從歷程屬性中的每個類別中選取適當的標籤。 歷程通常會在每個類別中包含一個標籤。

**步驟4 — 為里程碑命名歷程，依標籤篩選**

歷程名稱將焦點放在它所帶動的客戶里程碑上（例如&#x200B;*首次忠誠度交易*）。 在歷程清單中使用標籤篩選器，透過任何中繼資料組合來尋找歷程，而不依賴名稱剖析。

>
>如需深入討論此方法及其大規模優點，請參閱[Journey Optimizer中進階歷程的最佳實務](https://experienceleague.adobe.com/en/perspectives/best-practices-for-advanced-journeys-in-journey-optimizer){target="_blank"}。

**意圖：**
* 從歷程屬性標籤欄位新增標籤至歷程
* 依一或多個標籤篩選歷程清單，以快速找出特定歷程
* 編輯任何狀態的現有歷程上的標籤（即時、草稿等） 透過更多動作
* 以管理員身分建立並組織標籤類別，以強制實施一致的中繼資料
* 使用結構化的標籤式方法取代複雜的歷程命名慣例

**字彙表：**
* **標籤**：附加至歷程的標籤，以分類及篩選它們；不區分大小寫，且在複製歷程或為其建立版本時保留&#x200B;*（產品專屬）*
* **標籤類別**：系統管理員在「管理>標籤」下建立的相關標籤值群組，啟用結構化中繼資料分類&#x200B;*（產品特定）*
* **未分類的類別**：直接在歷程中建立標籤的內建預設類別會自動指派給&#x200B;*（產品特定）*

**護欄：**
* 標籤不區分大小寫
* 歷程中定義的標籤會自動新增到內建的「未分類」類別，除非管理員將其指派給已命名的類別
* 只有管理員可以透過「管理>標籤」選單刪除標籤及管理標籤類別
* 標籤會在歷程複製或建立新版本時保留

**術語：**
* 正式名稱：標籤 — 縮寫：無 — 變體：歷程標籤、管理標籤
* 正式名稱：標籤類別 — 縮寫：無 — 變體：標籤群組
* 請勿混淆：「標籤」（歷程分類標籤）≠「命名慣例」（直接在歷程名稱中編碼的中繼資料）

**常見問題集：**
* **問：如何將標籤新增至歷程？**  — 在歷程屬性中，在「標籤」欄位中輸入標籤名稱，並從清單中選取它，或按一下「建立」以新增標籤。
* **問：我可以新增標籤到即時歷程嗎？**  — 是。 按一下清單中歷程旁邊的更多動作圖示，然後選擇編輯標籤以在任何歷程上新增或移除標籤，無論狀態為何。
* **問：標籤是否區分大小寫？**  — 否。 標籤不區分大小寫。
* **問：當我複製歷程或建立新版本時，標籤會發生什麼事？**  — 標籤會保留在重複版本或新版本上。
* **問：誰可以刪除標籤或建立標籤類別？**  — 只有管理員可以透過「管理」 > 「標籤」功能表刪除標籤及管理標籤類別。
* **問：為何使用標籤類別來取代命名慣例？**  — 標籤類別可透過受控清單強制執行一致性、允許即時多維度篩選、保持歷程名稱簡短且以里程碑為中心，以及透過新增類別而不重寫命名規則而輕鬆縮放。
