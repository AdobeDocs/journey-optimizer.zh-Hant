# 開始應對忠誠度挑戰 {#get-started-loyalty-challenges}

>[!AVAILABILITY]
>
>此功能目前在&#x200B;**私人測試版**&#x200B;中，可能無法在您的環境中使用。 請聯絡您的 Adobe 代表以取得存取權。

忠誠度挑戰可讓您為客戶建立個人化的參與優惠方案，協助您大規模編排忠誠度計畫。 您可以設計具有特定任務和里程碑的挑戰、獎勵完成任務的客戶，並透過Adobe Journey Optimizer管道提供體驗。

**忠誠度挑戰檔案：**

* [瞭解忠誠度挑戰](get-started.md) — 功能、工作流程、必要條件
* [建立挑戰](create-challenges.md) — 建置並設定挑戰
* [管理挑戰](manage-challenges.md) — 編輯、監視、最佳化

>[!ENDSHADEBOX]

## 快速概覽 {#quick-overview}

使用忠誠度挑戰來：

* **建立三種型別的挑戰**：標準（任何工作）、條紋（重複的工作）或順序（排序的工作）
* **設計挑戰內容**：使用內容卡在客戶裝置上顯示挑戰
* **設定任務需求**：定義購買、造訪或具獎勵的自訂事件等動作
* **傳送多頻道通知**：透過應用程式內、電子郵件和關鍵階段的推播傳遞訊息
* **追蹤效能**：透過自動產生的歷程和內建報告進行監視

## 運作方式 {#how-it-works-overview}

依照此工作流程建立忠誠度挑戰：

1. **設定資料擷取** — 設定來源聯結器以追蹤客戶動作
2. **建立挑戰** — 定義型別、對象和日期
3. **新增任務** — 設定動作和獎勵
4. **設計內容** — 建立內容卡和選用的訊息
5. **發佈** - Journey Optimizer會自動產生並啟用歷程
6. **監視器** — 追蹤參與率和績效

>[!NOTE]
>
>自動產生的歷程會顯示在您的歷程詳細資料中，並可視需要自訂。 不過，直接對歷程進行的變更不會同步回挑戰設定。

## 先決條件 {#prerequisites-overview}

使用忠誠度挑戰之前：

* 設定Experience Platform來源聯結器（例如Chariceline）以擷取忠誠度事件資料
* 確定您在Journey Optimizer中有適當的許可權
* 在Experience Platform中建立目標對象

如需詳細的必要條件，請參閱[瞭解忠誠度挑戰](get-started.md#prerequisites)。

## 重要限制 {#limitations-overview}

* **沒有分類帳系統**：忠誠度挑戰不會追蹤貨幣值或點數餘額。 Journey Optimizer會呼叫您的外部忠誠度管理系統以處理點數配置。
* **僅限對象選擇**：您可以選取現有對象，但無法從「忠誠度挑戰」UI建立新對象。
